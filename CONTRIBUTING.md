# Contributing

Quick guidelines:

- Use Conventional Commits (feat/fix/chore/docs/etc).
- Add a short Unreleased entry to CHANGELOG.md for feature PRs.
- Run `flutter analyze` and `flutter test` before raising a PR.
- Use feature branches: `feat/...`, `fix/...`, `chore/...`.
- Use the PR template and templates for issues.

If you need to make releases:

- Releases are prepared on `main` after approved merges.
- Follow the release checklist in ROADMAP.md.
